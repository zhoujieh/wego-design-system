# 动态商品流

## 测试提示词

为微购相册设计一个动态商品流场景。

目标 让用户在微购相册中连续浏览店主发布的商品动态，每条动态稳定呈现发布者、发布时间、商品内容、商品图片与相关产品信息，并支持进入产品详情或执行必要操作。

纳入范围

动态商品流浏览场景：连续浏览店主发布的商品动态
每条动态稳定呈现：发布者、发布时间、商品内容、商品图片、相关产品信息
兼容只有产品信息 / 同时含文字 两种动态形态
兼容文字为空、图片数量不同、产品信息较长时的稳定展示
进入产品详情入口
必要操作入口：一键转发（核心操作）、复制文案、分享、收藏、下载图片、搜索、编辑、复制动态、图片点击看大图
排除范围

发布动态、评论点赞等社交互动
必要操作的具体功能实现（本轮只放入口，不下钻）
真实后端接口与持久化
入口（产品决策）

动态主 tab：作为底部导航「动态」tab 的内容，打开 App 即默认进入
关键路径

进入动态流 → 连续浏览动态 → 进入产品详情
进入动态流 → 连续浏览动态 → 执行必要操作
原型边界

动态商品流浏览场景 functional：可连续浏览、动态状态真实可操作
产品详情 simulated：本地模拟数据展示基本产品信息
必要操作入口 stub：本轮只呈现入口与明确反馈，不下钻具体功能实现

请根据微购设计系统完成该场景的产品分析、交互设计和可运行原型。
