# 文案与数据规范

## 标点符号

| 符号 | 用法 | 示例 |
|------|------|------|
| `、` 顿号 | 名词/动词/形容词并列 | 收款码、营销中心、铺货管家 |
| `/` 斜杠 | 表示选择（或） | 是/否 |
| `( )` 半角括号 | 中文/数字/英文中统一使用半角，不加空格 | 确认开单(3件) |
| `" "` 直角引号 | 引用固定名词、功能入口、路径 | 前往"营销中心" |
| `~` 波浪线 | 仅用于表示范围 | `20.00 ~ 30.00` |

## 文字着重

对需强调的内容进行加粗处理（Medium 500），使用 `#1E2028` 颜色，不使用品牌色高亮。

## 日期格式

| 规则 | 格式 | 示例 |
|------|------|------|
| 日期+时刻 | `YYYY/MM/DD HH:MM` | `2024/11/01 14:06` |
| 日期范围 | `~` 连接，前后空格 | `2022/06/21 ~ 06/25` |

### 年份省略规则

| 场景 | 规则 | 示例 |
|------|------|------|
| 过去日期 | 必须带年份 | `2024/01/01` |
| 今年内未来日期 | 省略年份 | `12/31` |
| 其他年份未来日期 | 必须带年份 | `2025/10/14` |
| 同年范围 | 结束日期省略年份 | `2022/06/21 ~ 06/25` |
| 跨年范围 | 首尾都带年份 | `2022/01/01 ~ 2024/01/01` |

## 时刻格式

| 规则 | 说明 |
|------|------|
| 24 小时制 | 晚上 7 点 = 19 点 |
| 统一两位 | 9 点 = `09:00` |
| 默认不显示秒 | 除非特殊要求 |

### 时间范围

| 场景 | 格式 | 示例 |
|------|------|------|
| 同天 | `HH:MM ~ HH:MM` | `20:15 ~ 20:25` |
| 跨 1 天 | 使用"次日" | `21:00 ~ 次日9:00` |
| 跨 2 天以上 | 带日期 | `06/21 21:00 ~ 06/23 9:00` |

## 过去时间表达

| 距今时长 | 表达 |
|----------|------|
| 0~59 秒 | `刚刚` |
| 1~59 分钟 | `N分钟前` |
| 1~23 小时 | `N小时前` |
| 1~1.99 天 | `昨天` |
| 2 天以上 | `YYYY年MM月DD日` |

## 数据格式

### 通用规则

| 规则 | 说明 |
|------|------|
| 不可折行 | 数字出现在行尾且无法完整显示时，另起一行 |
| 符号无空格 | `￥`、`+`、`-`、`%`、`/` 与数字间不加空格 |
| 最多两位小数 | 超出则四舍五入 |
| 范围格式 | `~` 连接，仅开头用货币符号 |
| 无数据 `--` | 数据异常、不适用（非人为制造时无意义） |
| 零数据 `0` | 正常统计但值为零 |

### 金额规则

| 规则 | 说明 |
|------|------|
| 强制两位小数 | `￥20.00`、`￥20.80` |
| 末尾零保留 | `20.80` 不省略 |

### 非金额规则

| 规则 | 说明 |
|------|------|
| 无小数不显示 | `20` 而非 `20.0` |
| 末尾零省略 | `20.8%` 而非 `20.80%` |

### 数字省略

- 达到 6 位数（>=10 万）可选择以"数字+万"省略
- 省略时四舍五入最多两位小数，末尾零省略（`12万`、`22.99万`、`23万`）
- 需要计算或修改的数字不应省略

### 商品价格特殊规则

- 使用 WegoKeyboard N9 字体
- 货币符号和小数部分小写，字号约为数字字号的 60%~75%（按数字字号分级）
- 数字 10~14px 时，小写与数字同号
- 范围的 `~` 不小写

## 简称

- 英文简称全大写：APP、VIP
- 代词大写：TA
- 注意：iPhone、iPad、iOS 本身就是如此拼写，勿改为全大写

## Emoji 使用

- 目的：引导关注、活跃气氛、操作暗示
- 一行最多 1 个 emoji
- 指向类 emoji 用于可点击操作的内容方向

## 文案精简原则

移动端屏幕空间有限，所有 UI 文案应遵循精简原则，确保用户快速理解核心信息：

| 场景 | 规则 | 示例 |
|------|------|------|
| 标题 | 用动词或短语直接表达核心动作或结果，避免长句 | "确认删除？" 而非 "您是否确认要执行删除操作？" |
| 正文 | 仅保留用户决策必需的信息，背景和细节交给上下文 | "删除后无法恢复" 而非 "请注意，删除操作执行后将无法恢复数据" |
| 按钮 | 用 2-4 字动词短语，避免名词或长句 | "删除"、"取消"、"知道了" 而非 "确认删除"、"取消操作" |
| 链接 | 用专有名词或简短词组，不加"点击查看"等引导语 | "服务协议" 而非 "点击查看服务协议" |
| 提示 | 优先用结构（图标、状态、分组）表达，不重复说明 | 避免在禁用项旁补"此项不可选"说明 |

精简不等于省略：用户决策必需的风险、时效、不可逆后果等信息必须保留，只删除冗余修饰词和重复表达。
