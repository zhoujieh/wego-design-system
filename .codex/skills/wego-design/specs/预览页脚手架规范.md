# 预览页脚手架规范

## 目的

为所有 `preview/component-*.html`、`preview/index.html` 和 `ui_kits/*.html` 提供统一的页面脚手架样式，确保预览页结构和视觉一致。

`scaffold.css` 只用于预览页和 UI Kit，**不用于生产组件**。

## CSS 导入顺序

预览页 `<head>` 中按以下顺序引入：

1. `colors_and_type.css` -- Token 权威源
2. `scaffold.css` -- 脚手架样式
3. `iconfont.css` -- 图标字体
4. `components.css`（可选）-- 聚合组件样式

```html
<link rel="stylesheet" href="../colors_and_type.css">
<link rel="stylesheet" href="../scaffold.css">
<link rel="stylesheet" href="../iconfont.css">
```

## 脚手架选择器

| 选择器 | 用途 |
|--------|------|
| `*` | 全局 box-sizing: border-box |
| `html, body` | 页面底色、字体、最小高度 |
| `body` | 32px 页面内边距 |
| `.pv-header` | 页面头部区域（标题 + 副标题） |
| `.pv-header h1` | 页面主标题，18px Semibold |
| `.pv-header p` | 页面副标题，14px 二级文字 |
| `.pv-section` | 分区容器，底部间距 32px |
| `.pv-section__title` | 分区标题，14px Semibold |
| `.pv-section__sub` | 分区副标题/说明，12px 三级文字 |
| `.row` | 水平 flex 布局，8px gap |
| `.label` | 左侧标签，12px 三级文字，固定 92px 宽 |
| `.stack-8 / .stack-12 / .stack-16 / .stack-24` | 纵向 flex 布局，对应间距 |
| `.page` | 手机列表页容器，最大宽度 420px 居中 |
| `.section-group` | 列表页分组，白色背景（预览脚手架级容器，仅供 preview/ 和 ui_kits/ 使用；任务产物中成组 cell 必须用组件契约层的 `.cell-group`，不用 `.section-group`） |
| `.section-gap` | 分组间间隙，8px 灰色背景 |
| `.preview-phone` | 手机预览外壳，375px 宽，圆角 + 轻阴影 |
| `.preview-phone__content` | 手机外壳内容区 |
| `.dark-strip` | 暗色展示条，用于暗色背景下的组件展示 |

## 安全区规则

适用于所有手机模拟页面，包括：

- `ui_kits/*.html` 中的 UI Kit 演示页
- `wego-ux/templates/*.html` 中的任务产物模板（page-shell.html、host-shell.html）
- 后续生成的任务级原型中的手机模拟页面（`.phone-screen` 内的内容）

### 核心原则

安全区处理采用"**状态栏悬浮 + 内容自身留距**"模式：
- phone-status / phone-indicator 绝对定位悬浮，透明背景，不占布局空间
- phone-status / phone-indicator 在预览外壳内保持最高层级，统一使用 `z-index: var(--z-critical)`
- 内容通过自身的 padding-top / padding-bottom 避开安全区
- **不要给父屏幕容器加 padding-top / padding-bottom**（会导致绝对定位的状态栏/指示器位置错乱）

### 顶部状态栏（phone-status）

- `position: absolute; top: 0` 悬浮在屏幕最顶部，不占布局空间
- 背景透明，由下方导航栏/页面背景透出颜色
- 高度由 `--safe-area-top` Token 控制（桌面浏览器 env() 返回 0px，真机返回设备实际值；`.phone-screen` 内提供 44px 模拟值，移动端 @media 重置为 0）
- navbar 在组件内部通过 `padding-top: var(--safe-area-top, 0px)` + sticky `top: 0` 自动避让顶部安全区（components.css 已统一配置，宿主无需额外处理）
- navbar 主体高度由 `--navbar-body-height` 控制：默认 `56px`；桌面手机壳显示 `phone-status` 时，`.phone-screen` 内覆盖为 `44px`；移动端同链接隐藏状态栏后重置回 `56px`
- 深色/图片背景场景使用 `.navbar--fixed-transparent` 修饰类切换为 fixed 透明反白模式，此时 page-body 需加 `padding-top: calc(var(--navbar-body-height, 56px) + var(--safe-area-top, 0px))` 为 navbar 与顶部安全区让位
- 任务产物模板（page-shell.html）和宿主模板（host-shell.html）的 `.phone-status` 同样遵循此 absolute 悬浮方案；`page.css` / `host-shell.css` 在 `.phone-screen` 内定义 Token 模拟值，移动端隐藏预览状态栏/指示器并改由真机 `env(safe-area-inset-*)` 返回真实安全区

**正确结构**：
```html
<div class="screen">
  <div class="phone-status"></div>  <!-- 绝对定位，top:0 -->
  <div class="navbar"></div>  <!-- 组件内部 padding-top safe-area，sticky top:0 吸顶 -->
  <div class="phone-body"></div>
</div>
```

**常见错误（不要这么做）**：
```css
/* ❌ 错误：给父容器加 padding-top，会导致绝对定位的 status bar 往下偏移 */
.screen { padding-top: var(--safe-area-top); }
```

### 底部 Home 指示器（phone-indicator）

- `position: absolute; bottom: 0` 悬浮在屏幕最底部，不占布局空间
- 背景透明
- 高度由 `--safe-area-bottom` Token 控制（桌面浏览器 env() 返回 0px，真机返回设备实际值；`.phone-screen` 内提供 34px 模拟值，移动端 @media 重置为 0）
- 任务产物模板（page-shell.html）和宿主模板（host-shell.html）的 `.phone-indicator` 同样遵循此 absolute 悬浮方案
- 可滚动内容（page-body / host-shell-page）底部通过 `padding-bottom: calc(var(--spacer-40) + var(--safe-area-bottom))` 避开安全区

### 底部固定元素 safe-area 让位

页面底部固定元素（bottom-nav、固定操作栏、固定按钮栏等）必须预留底部安全高度间距，规则：

- 固定元素底部 padding 使用 `padding-bottom: var(--safe-area-bottom, env(safe-area-inset-bottom, 0px))`（Token 优先、env 兜底）
- 已注册组件（如 bottom-nav）的 safe-area padding 由 `components.css` 统一提供，宿主和任务级 `page.css` 不重复覆盖
- 任务级自创固定操作栏（如底部保存按钮栏、底部结算栏）必须在 `page.css` 业务样式中显式声明 `padding-bottom: var(--safe-area-bottom, env(safe-area-inset-bottom, 0px))`
- 若页面底部没有固定操作栏，则可滚动内容容器（page-body / host-shell-page）底部必须补 `calc(40px + var(--safe-area-bottom))`
- 移动端 @media 通过在 `.phone-screen` 重置 `--safe-area-bottom: 0` 联动归零所有依赖样式，不逐个重置固定元素 padding

### 安全区 Token

```css
/* 由 .phone-screen 提供，桌面外壳模拟手机安全区 */
--safe-area-top: 44px;
--safe-area-bottom: 34px;
--navbar-body-height: 44px;

/* 移动端 @media (max-width: 767px) 内重置 safe-area 为 0，并把 navbar 主体高度恢复为默认 56px */
.phone-screen {
  --safe-area-top: 0;
  --safe-area-bottom: 0;
  --navbar-body-height: 56px;
}
```

`env(safe-area-inset-*)` 由真机 WebView 返回实际值；桌面外壳通过 `.phone-screen` 内显式定义 Token 模拟值代替 env()，移动端同链接通过 @media 重置 Token 为 0 实现联动隐藏。

## 布局模式

### 模式 A：通用预览（适用于大多数组件）

使用 `pv-header` + `pv-section` + `.row` + `.label` 组织内容。

```html
<div class="pv-header">
  <h1>组件名称</h1>
  <p>简要描述</p>
</div>

<div class="pv-section">
  <div class="pv-section__title">分区标题</div>
  <div class="pv-section__sub">分区说明</div>
  <div class="row">
    <span class="label">标签</span>
    <!-- 组件实例 -->
  </div>
</div>
```

### 模式 B：列表模拟（适用于 cell、form 等列表型组件）

使用 `.page` + `.section-group` + `.section-gap` 模拟移动端列表页面。

```html
<div class="page">
  <div class="section-group">
    <!-- 列表组件实例 -->
  </div>
  <div class="section-gap"></div>
  <div class="section-group">
    <!-- 另一组列表 -->
  </div>
</div>
```

### 模式 C：聚合验收入口（适用于 `preview/index.html`）

允许使用脚手架区块 + 自定义卡片容器聚合多个单组件预览页，推荐通过 `iframe` 内嵌 `component-*.html`，用于人工验收集中查看。

- 聚合页只负责入口和排布，不复制组件核心示例。
- 聚合页必须明确标注“仅供人工验收查看”，避免被误当成组件契约入口。
- 新增已注册组件时，必须同步把该组件加入聚合页。

## 组件 CSS 标记规则

预览页 `<style>` 块中，组件样式必须用提取标记包裹：

```css
/* @component-css-start */
.tag { ... }
.tag--danger { ... }
/* @component-css-end */
```

- 标记之间只放**组件核心 CSS**。
- 脚手架样式（`.row`、`.label`、`.page`、`.section-group`、`.demo-hint` 等）必须在标记**之外**。
- 脚手架选择器已由 `scaffold.css` 统一提供，预览页不应重复定义。
- `preview/index.html` 这类聚合页不得放 `@component-css-start` / `@component-css-end` 标记，避免被提取脚本误收。

## 使用约束

- `scaffold.css` 仅限 `preview/` 和 `ui_kits/` 目录下的 HTML 文件使用。
- 生产组件不引用 `scaffold.css`，组件样式通过 `components.css` 或内联方式提供。
- 新建预览页时，必须引入 `scaffold.css`，不再自行定义脚手架样式。
- `preview/index.html` 仅作为人工验收聚合入口，不替代 `preview/component-*.html` 单组件预览页。
