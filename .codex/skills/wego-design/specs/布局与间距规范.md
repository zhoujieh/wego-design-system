# 布局与间距规范

## 核心原则

**4N 规则**：所有元素尺寸和间距尽量取 4 的倍数，确保视觉节奏统一。

## 页面边距

页面留白由边距控制，而非列表内边距（列表内边距固定 16px）。

| 类型 | 左右边距 | 留白程度 | 适用场景 |
|------|----------|----------|----------|
| M1 | 0px | 无留白 | 通栏内容 |
| M2 | 16px | 中等留白 | 常规卡片（默认） |
| M3 | 32px | 大留白 | 宽松卡片、表单 |

### 模式选择规则（M1/M2）

| 模式 | 页面横向 padding | cell-group__content | 适用场景 |
|------|------------------|---------------------|----------|
| 通栏 M1 | 0px | 无 --card 修饰 | biz-rule-config 命中页面（业务规则配置、权限管理、库存规则等） |
| 卡片 M2 | 16px | 开 --card 修饰 | host-entry 宿主入口列表、常规卡片页 |

- **通栏 M1 的内容横向边距**由 `cell__body` / `form-body` 自带 16px padding 承担，cell 内容不会贴边。
- **卡片 M2 的内容横向边距**由 phone-body 16px padding + cell-group__content--card 圆角白底共同承担。
- 命中 `biz-rule-config` pagePattern 的 surface 默认通栏 M1；仅当 surface 角色为 `host-entry` 时才改为卡片 M2。
- 移动端与桌面端的页面横向 padding 必须一致（M1=0px 或 M2=16px），不得因移动端 phone-frame border 消失而调整 scene body padding。
- **phone-body 内容上间距**：统一使用 `padding-top: var(--spacer-8)`（8px），确保内容区域与上方导航栏之间有一致的呼吸空间。

## 页面背景### 移动端横向 padding 适配规则（host-shell-page 护栏）

宿主容器 `.host-shell-page` 在桌面端使用 3 值 padding 简写 `padding: <top> <left/right> <bottom>`，其中 top/bottom 依赖 `.phone-screen` 提供的 `--safe-area-top` / `--safe-area-bottom` 变量。移动端 `@media (max-width: 767px)` 把这两个变量重置为 `0` 时，3 值简写在 CSS 变量级联场景下可能整条声明失效，导致横向 padding 跌落到 `0`。为防止此问题，必须遵守以下规则：

- `.host-shell-page` 必须在移动端媒体查询 `@media (max-width: 767px)` 中显式声明 `padding-left` / `padding-right`，禁止只依赖桌面端的 3 值 padding 简写
- 移动端横向 padding 取值应与桌面端一致（M1=0px，M2=16px），或使用 `calc(var(--spacer-16) + env(safe-area-inset-left/right, 0px))` 支持刘海屏横向安全区
- viewport meta 必须包含 `viewport-fit=cover`，否则 `env(safe-area-inset-*)` 在 iOS 上始终返回 0，env() 形同虚设
- 模板 `host-shell.css` / `host-shell.html` 与产物 `wego-app/css/app.css` / `wego-app/index.html` 必须保持同步更新，避免下次 AI 生成新 app 时问题复现

## 页面背景

页面背景色决定页面整体基调，导航栏背景色必须与页面背景色保持一致（显式设置，不使用 transparent）。

| 背景类型 | Token | 适用场景 |
|----------|-------|----------|
| 灰色页面 | `--bg-page` | 列表页、设置页、表单页（内容区块用白色卡片承载） |
| 白色页面 | `--bg-surface` | 详情页、空白页、入口页、登录注册页 |

- 每个页面单独定义背景色，不做二级页面继承约束。
- 导航栏必须显式设置背景色，与页面 `data-bg` 声明的背景色一致；不要使用 `transparent`（深色场景除外）。
- 页面通过 `data-bg` 属性声明背景类型（`surface` = 白 / `page` = 灰），用于组件自适应。

## 页面外层容器

页面外层容器约束最大宽度，内容区域宽度 100%。

| 属性 | 值 | 说明 |
|------|-----|------|
| `max-width` | `768px` | 外层容器最大宽度，居中显示 |
| `margin-inline` | `auto` | 水平居中 |
| 内容宽度 | `100%` | 内容区域始终撑满外层容器，不写死固定宽度 |

- 所有页面内容（含导航栏）宽度默认 100%，由外层容器的 `max-width: 768px` 约束。
- 不要给组件或内容区写死固定宽度（如 375px）；固定宽度只允许出现在预览脚手架中。
- 沉浸页（如全屏图片浏览）可去掉 `max-width` 约束，由业务自行处理。

## 纵向分组

| 类型 | 组内间距 | 组间间距 | 排列风格 |
|------|----------|----------|----------|
| G1 | 0px | 16px | 紧密 |
| G2 | 8px | 32px | 宽松 |

## 辅助说明文字间距

列表组底部的辅助说明文字（helper text）需要同时考虑纵向间距和横向对齐，不能只处理单一方向。

### 完整盒模型

| 属性 | 值 | 说明 |
|------|-----|------|
| `margin-top` | `12px` | 与上方列表组的纵向间距，用 margin 而非 padding，确保与上方白色卡片之间有清晰的背景色边界 |
| `padding` | `12px 16px 20px` | 内容内边距：上 12px、左右 16px、下 20px；左右 16px 与列表/表单内容的左右内边距保持一致，文字左对齐 |
| `width` | 通栏 | 和列表组同宽，不单独设置左右 margin |

### 关键规则

- **四方向都要考虑**：处理间距时必须同时检查上下左右，不能只补一个方向就结束。
- **纵向用 margin，横向用 padding**：纵向与上方内容的分隔靠 `margin-top`（背景色断开），横向内容缩进靠 `padding-left/right`（和列表内容对齐）。
- **左对齐基线**：helper text 的文字左缘必须与上方列表/表单的文字左缘对齐，不能出现列表内容缩进 16px 而说明文字贴边的情况。
- **右对齐同理**：右侧也要保持一致的 16px 内边距，长文本换行后右侧不贴屏幕边缘。

## 列表与表单

- 内边距：16px
- 左右信息组间距：24px

## 卡片

| 类型 | 内边距 | 适用场景 |
|------|--------|----------|
| 默认容器 | 0px | 组件根节点；优先把点击热区留给内容本身 |
| 常规内容区 | 16px | 卡片内部标准内容分区 |
| 宽松内容区 | 32px | 需要更多呼吸空间的内容分区 |

卡片默认不在根节点写内边距，内容留白由内部内容区或业务内容自行处理。只有当同一组内容需要统一留白时，才在内容分区上加内边距。

## 信息组间距

### 左右布局：图标 + 文字

图标与文字表达相同含义时的间距：

| 文字尺寸 | 图标尺寸 | 间距 |
|----------|----------|------|
| 16px | 20px | 4px |
| 14px | 16px | 4px |
| 12px | 14px | 2px |
| 10px | 12px | 0px |

### 左右布局：文字 + 操作图标

文字与操作图标（跳转/切换）之间无间距。

### 上下布局：图标 + 文字

| 图标尺寸 | 文字尺寸 | 场景 |
|----------|----------|------|
| 24px | 10px | 底部 Tab |
| 20px | 10px | 应用入口 |

### 辅助说明

| 组合 | 说明 | 间距 |
|------|------|------|
| 主文本16 + 辅助12 | 横向或纵向 | 4px |
| 主文本18 + 辅助14 | 横向或纵向 | 4px |
| 按钮48 + 文本12 | 纵向 | 8px |

### 标题 + 内容

| 标题尺寸 | 内容尺寸 | 间距 |
|----------|----------|------|
| 18px | 14px | 8px |
| 22px | 14px | 8px |

## 操作热区

### 核心原则

1. **热区不破坏布局**：视觉尺寸归视觉，热区归热区。需要补偿时只通过伪元素扩展，绝对不要为了热区而放大组件的布局占用。
2. **只处理过小目标**：仅当点击目标视觉尺寸小于 `32px` 时，才考虑补充热区；`32px` 及以上默认不强制扩展。
3. **仅扩展可操作节点**：只给 `button`、`a`、带 `onclick` / `cursor:pointer` 的可交互元素扩展热区，不对纯展示元素扩展。

### 热区参考体系

移动端仍保留三档热区 Token 作为参考值，但它们不是一刀切的强制最小尺寸。是否扩展，优先看点击目标是否已经达到 `32px`，以及扩展后是否会破坏布局。

| 层级 | Token | 最小尺寸 | 适用场景 |
|------|-------|---------|----------|
| 紧凑 | `--touch-min` | 40 × 40px | 信息密集型列表中的小操作项（如 cell 内嵌按钮、chip、行内图标操作） |
| 默认 | `--touch-default` | 44 × 44px | 常规操作元素（如 icon-button、文字按钮、底部导航 Tab） |
| 舒适 | `--touch-comfortable` | 48 × 48px | 主操作按钮、独立大按钮、导航栏操作区 |

### 热区扩展方法：伪元素技术

当元素视觉尺寸小于 `32px`，且确实需要补充可点击范围时，通过 `::before` 或 `::after` 伪元素扩展触控区域。伪元素使用绝对定位向外延伸，**不占布局空间、不影响文档流**。

**基础模板**：

```css
/* 父元素需设置 position: relative（如未设置则需加上） */
.touch-target {
  position: relative;
}
/* 向四周扩展 */
.touch-target::before {
  content: '';
  position: absolute;
  /* 计算方式：(最小热区 - 视觉尺寸) / 2，向四边各扩展该值 */
  /* 例：视觉 28px → 热区 44px，每边扩展 (44-28)/2 = 8px */
  top: -8px;
  right: -8px;
  bottom: -8px;
  left: -8px;
}
/* 仅向缺少的方向扩展（视觉宽度够但高度不够时） */
.touch-target::before {
  content: '';
  position: absolute;
  /* 水平够、高度不够，只扩展上下 */
  top: -6px;
  bottom: -6px;
}
```

### 按元素类型分场景

#### 图标按钮（icon-button）

视觉尺寸通常 16~24px，在列表、工具栏、导航栏中高频出现。

| 场景 | 视觉尺寸 | 是否建议扩展 | 扩展方式 |
|------|---------|--------------|---------|
| 导航栏操作图标 | 24 × 24px | 是 | `::before` 四周补偿，但不挤占导航栏布局 |
| 列表行内图标操作 | 16 × 16px | 是 | `::before` 四周补偿，必要时仅补单侧 |
| 底部导航 Tab 图标 | 24 × 24px | 否 | 自身 Tab 区域已足够，不扩展 |
| 卡片角标关闭 | 12 × 12px | 是 | `::before` 四周补偿 |
| 输入框清除按钮 | 16 × 16px | 是 | `::before` 四周补偿 |

#### 文字按钮（text-button）

| 场景 | 视觉尺寸 | 是否建议扩展 | 扩展方式 |
|------|---------|--------------|---------|
| 导航栏文字操作 | 约 56 × 28px | 是 | 高度小于 32px，可仅上下扩展 |
| 列表右侧文字操作 | 约 40 × 20px | 是 | 高度小于 32px，可仅上下扩展 |
| 独立文字按钮 | ≥ 32px 高 | 否 | 自身已满足，不扩展 |

#### 色块按钮（filled button）

色块按钮的 `padding` 通常已经保证可点范围。默认不再因为未达到某个绝对热区值而强制扩展；只有点击目标小于 `32px` 时才补偿。

| 场景 | 视觉尺寸（含 padding） | 是否建议扩展 | 说明 |
|------|----------------------|--------------|------|
| 小按钮 sm | 约 60 × 32px | 否 | 已达到 32px，下限内不强制扩展 |
| 中按钮 md | ≥ 40px 高 | 否 | 自身已满足，不扩展 |
| 大按钮 lg | ≥ 48px 高 | 否 | 自身已满足，不扩展 |
| 底部固定按钮 | 全宽 × 48px | 否 | 自身已满足，不扩展 |

#### 列表单元格（cell）

cell 整体通常已超过热区：`16px padding-top + 内容 + 16px padding-bottom ≥ 48px`。cell 内嵌的可操作子元素（按钮、图标、chip）才需要单独扩展热区。

#### 计数器（counter）

计数器按钮视觉尺寸 28 × 28px，组件视觉高度 32px。因为点击目标小于 `32px`，可使用 `counter__btn::before` 做不对称补偿，保证按钮可点击而不改变 32px 的组件高度。

#### 表单控件（checkbox / radio / switch）

| 组件 | 视觉尺寸 | 是否建议扩展 | 说明 |
|------|---------|--------------|------|
| checkbox (24px) | 24 × 24px | 是 | 可用 `::before` 四周补偿 |
| checkbox (20px) | 20 × 20px | 是 | 可用 `::before` 四周补偿 |
| radio (24px) | 24 × 24px | 是 | 可用 `::before` 四周补偿 |
| radio (20px) | 20 × 20px | 是 | 可用 `::before` 四周补偿 |
| switch | 52 × 32px | 否 | 已达到 32px 高，默认不扩展 |

#### 底部导航（bottom-nav）

每个 Tab 的自身布局区域已 ≥ 默认热区，无需伪元素扩展。但视觉图标仅占 24px，Tab 自身的 padding 已补足触控空间。

### 热区冲突处理

当多个可操作元素的热区可能重叠时，处理优先级：

1. 优先通过布局间距避免冲突（元素间距 ≥ 8px）。
2. 如不可避（如密集列表），将较小/次要操作的热区限定在布局范围内，不做全方向扩展（仅补足高度或单侧）。
3. 相邻操作元素使用 `z-index` 或 DOM 顺序保证主要操作优先响应。

### Token 参考

```css
/* 来自 colors_and_type.css */
--touch-min: 40px;        /* 紧凑热区 */
--touch-default: 44px;    /* 默认热区 */
--touch-comfortable: 48px; /* 舒适热区 */
```

### 自检清单

设计可操作元素时，确认：

- [ ] 点击目标是否小于 32px？若否，默认无需额外处理。
- [ ] 若小于 32px，是否用伪元素补充了热区？
- [ ] 伪元素扩展是否不影响文档流和相邻元素布局？
- [ ] 父元素是否设置了 `position: relative`（伪元素依赖）？
- [ ] 相邻操作元素热区是否有冲突？是否按优先级处理？
