# 交互设计原则

## 操作布局原则

- 主要操作（保存、完成）放右侧：右上角或右下角
- 确定性/前进性按钮放右侧
- 中断/返回性按钮放左侧
- 主要操作与风险操作分开放置，避免误操作

## 交互状态定义

| 状态 | Token 后缀 | 适用平台 | 说明 |
|------|-----------|----------|------|
| 普通 | `-normal` / 默认 | 全平台 | 默认外观 |
| 悬浮 | `-hover` | PC/网页端 | 光标变手指，80% 不透明度 |
| 点击 | `-onclick` / `:active` | 全平台 | 60% 不透明度 |
| 禁用 | `-disabled` | 全平台 | 20% 不透明度，禁止使用失效色 |

## 页面模式与转场

移动端页面按呈现方式分为两种模式：**全屏页面**和**底部面板**。

### 全屏页面（Fullscreen）

- **特点**：占据整个屏幕，替换当前页的全部内容区域
- **适用场景**：主流程页面（首页、商品详情、我的）
- **导航方式**：通过底部 Tab 切换，或从列表页点击进入子页面
- **返回方式**：顶部导航栏返回按钮，或底部 Tab 切换

### 底部面板模式（Bottom Sheet）

- **触发**：从当前页面底部向上滑入，覆盖在原有内容之上
- **适用场景**：设置页、筛选页、选择器、详情操作面板等辅助流程
- **层级**：`--z-overlay` (500)，覆盖当前页面内容，但保留底部 Tab 可见
- **关闭**：点击取消/保存按钮关闭，或通过下拉手势关闭
- **动画**：`--ease-enter`，`--duration-normal` (250ms)，从底部向上滑入
- **典型页面**：岗位权限设置、业务配置、商品筛选、客服聊天

### 底部面板的实现要点

1. **导航栏**：使用 `navbar` 组件，左侧放置"取消"文字按钮，中间标题居中，右侧放置"保存"按钮
2. **内容区**：主体使用 `cell` 组件构建列表，`section-group` 包裹实现白色背景和分割线
3. **控件选择**：
   - 布尔开关：`switch` 组件
   - 多选子项：`checkbox` 组件嵌套在 `cell__select` 内
   - 跳转选择：`cell__action-text` + `icon-xiajiantou16` 指示器
4. **分组间距**：不同功能组之间用 `section-gap`（8px 灰色间隔）分隔
5. **保持底部 Tab**：底部面板模式下，底部导航栏（`bottom-nav`）保持可见不被遮挡
