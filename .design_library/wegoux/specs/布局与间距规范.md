# 布局与间距规范

## 核心原则

**4N 规则**：所有元素尺寸和间距尽量取 4 的倍数，确保视觉节奏统一。

## 页面边距

页面留白由边距控制，而非列表内边距（列表内边距固定 16px）。

| 类型 | 左右边距 | 留白程度 | 适用场景 |
|------|----------|----------|----------|
| M1 | 0px | 无留白 | 通栏内容 |
| M2 | 16px | 中等留白 | 常规卡片（默认） |
| M3 | 32px | 大留白 | 宽松卡片、表单 |

## 纵向分组

| 类型 | 组内间距 | 组间间距 | 排列风格 |
|------|----------|----------|----------|
| G1 | 0px | 16px | 紧密 |
| G2 | 8px | 32px | 宽松 |

## 列表与表单

- 内边距：16px
- 左右信息组间距：24px

## 卡片

| 类型 | 内边距 | 适用场景 |
|------|--------|----------|
| 普通 | 16px | 默认卡片 |
| 宽松 | 32px | 需要更多呼吸空间的卡片 |

卡片内部元素与四角距离相同。

## 信息组间距

### 左右布局：图标 + 文字

图标与文字表达相同含义时的间距：

| 文字尺寸 | 图标尺寸 | 间距 |
|----------|----------|------|
| 16px | 20px | 4px |
| 14px | 16px | 4px |
| 12px | 14px | 2px |
| 10px | 12px | 0px |

### 左右布局：文字 + 操作图标

文字与操作图标（跳转/切换）之间无间距。

### 上下布局：图标 + 文字

| 图标尺寸 | 文字尺寸 | 场景 |
|----------|----------|------|
| 24px | 10px | 底部 Tab |
| 20px | 10px | 应用入口 |

### 辅助说明

| 组合 | 说明 | 间距 |
|------|------|------|
| 主文本16 + 辅助12 | 横向或纵向 | 4px |
| 主文本18 + 辅助14 | 横向或纵向 | 4px |
| 按钮48 + 文本12 | 纵向 | 8px |

### 标题 + 内容

| 标题尺寸 | 内容尺寸 | 间距 |
|----------|----------|------|
| 18px | 14px | 8px |
| 22px | 14px | 8px |

## 操作热区

### 核心原则

1. **热区不破坏布局**：视觉尺寸归视觉，热区归热区。当元素视觉尺寸小于所需热区时，用伪元素扩展热区；绝对不要为了热区而放大组件的布局占用。
2. **够大则不自增**：元素自身可感知区域（含内边距）已大于等于对应层级热区时，热区即自身区域，不额外扩展。
3. **仅扩展可操作节点**：只给 `button`、`a`、带 `onclick` / `cursor:pointer` 的可交互元素扩展热区，不对纯展示元素扩展。

### 分层热区体系

移动端可操作元素按场景和信息密度分为三个层级，分别对应三个 CSS 变量：

| 层级 | Token | 最小尺寸 | 适用场景 |
|------|-------|---------|----------|
| 紧凑 | `--touch-min` | 40 × 40px | 信息密集型列表中的小操作项（如 cell 内嵌按钮、chip、行内图标操作） |
| 默认 | `--touch-default` | 44 × 44px | 常规操作元素（如 icon-button、文字按钮、底部导航 Tab） |
| 舒适 | `--touch-comfortable` | 48 × 48px | 主操作按钮、独立大按钮、导航栏操作区 |

### 热区扩展方法：伪元素技术

当元素视觉尺寸小于层级热区时，通过 `::before` 或 `::after` 伪元素扩展触控区域。伪元素使用绝对定位向外延伸，**不占布局空间、不影响文档流**。

**基础模板**：

```css
/* 父元素需设置 position: relative（如未设置则需加上） */
.touch-target {
  position: relative;
}
/* 向四周扩展 */
.touch-target::before {
  content: '';
  position: absolute;
  /* 计算方式：(最小热区 - 视觉尺寸) / 2，向四边各扩展该值 */
  /* 例：视觉 28px → 热区 44px，每边扩展 (44-28)/2 = 8px */
  top: -8px;
  right: -8px;
  bottom: -8px;
  left: -8px;
}
/* 仅向缺少的方向扩展（视觉宽度够但高度不够时） */
.touch-target::before {
  content: '';
  position: absolute;
  /* 水平够、高度不够，只扩展上下 */
  top: -6px;
  bottom: -6px;
}
```

### 按元素类型分场景

#### 图标按钮（icon-button）

视觉尺寸通常 16~24px，在列表、工具栏、导航栏中高频出现。

| 场景 | 视觉尺寸 | 热区层级 | 扩展方式 |
|------|---------|---------|---------|
| 导航栏操作图标 | 24 × 24px | 默认 44px | `::before` 四周扩展 10px |
| 列表行内图标操作 | 16 × 16px | 紧凑 40px | `::before` 四周扩展 12px |
| 底部导航 Tab 图标 | 24 × 24px | 默认 44px | 自身 Tab 区域已 ≥ 44px，不扩展 |
| 卡片角标关闭 | 12 × 12px | 紧凑 40px | `::before` 四周扩展 14px |
| 输入框清除按钮 | 16 × 16px | 紧凑 40px | `::before` 四周扩展 12px |

#### 文字按钮（text-button）

| 场景 | 视觉尺寸 | 热区层级 | 扩展方式 |
|------|---------|---------|---------|
| 导航栏文字操作 | 约 56 × 28px | 默认 44px | 高度不够，`::before` 上下各扩展 8px |
| 列表右侧文字操作 | 约 40 × 20px | 紧凑 40px | 高度不够，`::before` 上下各扩展 10px |
| 独立文字按钮 | ≥ 44 × 44px | 默认 44px | 自身已满足，不扩展 |

#### 色块按钮（filled button）

色块按钮的 `padding` 通常已经保证热区。

| 场景 | 视觉尺寸（含 padding） | 热区层级 | 说明 |
|------|----------------------|---------|------|
| 小按钮 sm | 60 × 32px | 默认（高不够） | 高度 32 < 44，用 `::before` 上下扩展 6px |
| 中按钮 md | ≥ 88 × 44px | 默认 44px | 自身已满足，不扩展 |
| 大按钮 lg | ≥ 全宽 × 48px | 舒适 48px | 自身已满足，不扩展 |
| 底部固定按钮 | 全宽 × 48px | 舒适 48px | 自身已满足，不扩展 |

#### 列表单元格（cell）

cell 整体通常已超过热区：`16px padding-top + 内容 + 16px padding-bottom ≥ 48px`。cell 内嵌的可操作子元素（按钮、图标、chip）才需要单独扩展热区。

#### 计数器（counter）

计数器按钮视觉尺寸 28 × 28px，组件视觉高度 32px。使用 `counter__btn::before` 不对称扩展热区至 40px，保证按钮可点击而不改变 32px 的组件高度。

#### 表单控件（checkbox / radio / switch）

| 组件 | 视觉尺寸 | 热区层级 | 说明 |
|------|---------|---------|------|
| checkbox (24px) | 24 × 24px | 紧凑 40px | `::before` 四周扩展 8px |
| checkbox (20px) | 20 × 20px | 紧凑 40px | `::before` 四周扩展 10px |
| radio (24px) | 24 × 24px | 紧凑 40px | `::before` 四周扩展 8px |
| radio (20px) | 20 × 20px | 紧凑 40px | `::before` 四周扩展 10px |
| switch | 52 × 32px | 紧凑 40px | 高度不足，`::before` 上下各扩展 4px |

#### 底部导航（bottom-nav）

每个 Tab 的自身布局区域已 ≥ 默认热区，无需伪元素扩展。但视觉图标仅占 24px，Tab 自身的 padding 已补足触控空间。

### 热区冲突处理

当多个可操作元素的热区可能重叠时，处理优先级：

1. 优先通过布局间距避免冲突（元素间距 ≥ 8px）。
2. 如不可避（如密集列表），将较小/次要操作的热区限定在布局范围内，不做全方向扩展（仅补足高度或单侧）。
3. 相邻操作元素使用 `z-index` 或 DOM 顺序保证主要操作优先响应。

### Token 参考

```css
/* 来自 colors_and_type.css */
--touch-min: 40px;        /* 紧凑热区 */
--touch-default: 44px;    /* 默认热区 */
--touch-comfortable: 48px; /* 舒适热区 */
```

### 自检清单

设计可操作元素时，确认：

- [ ] 视觉尺寸是否已达到对应层级热区？若已满足，无需额外处理。
- [ ] 若未满足，是否用伪元素扩展了热区？
- [ ] 伪元素扩展是否不影响文档流和相邻元素布局？
- [ ] 父元素是否设置了 `position: relative`（伪元素依赖）？
- [ ] 相邻操作元素热区是否有冲突？是否按优先级处理？
