# 预览页脚手架规范

## 目的

为所有 `preview/component-*.html`、`preview/index.html` 和 `ui_kits/*.html` 提供统一的页面脚手架样式，确保预览页结构和视觉一致。

`scaffold.css` 只用于预览页和 UI Kit，**不用于生产组件**。

## CSS 导入顺序

预览页 `<head>` 中按以下顺序引入：

1. `colors_and_type.css` -- Token 权威源
2. `scaffold.css` -- 脚手架样式
3. `iconfont.css` -- 图标字体
4. `components.css`（可选）-- 聚合组件样式

```html
<link rel="stylesheet" href="../colors_and_type.css">
<link rel="stylesheet" href="../scaffold.css">
<link rel="stylesheet" href="../iconfont.css">
```

## 脚手架选择器

| 选择器 | 用途 |
|--------|------|
| `*` | 全局 box-sizing: border-box |
| `html, body` | 页面底色、字体、最小高度 |
| `body` | 32px 页面内边距 |
| `.pv-header` | 页面头部区域（标题 + 副标题） |
| `.pv-header h1` | 页面主标题，18px Semibold |
| `.pv-header p` | 页面副标题，14px 二级文字 |
| `.pv-section` | 分区容器，底部间距 32px |
| `.pv-section__title` | 分区标题，14px Semibold |
| `.pv-section__sub` | 分区副标题/说明，12px 三级文字 |
| `.row` | 水平 flex 布局，8px gap |
| `.label` | 左侧标签，12px 三级文字，固定 92px 宽 |
| `.stack-8 / .stack-12 / .stack-16 / .stack-24` | 纵向 flex 布局，对应间距 |
| `.page` | 手机列表页容器，最大宽度 420px 居中 |
| `.section-group` | 列表页分组，白色背景 |
| `.section-gap` | 分组间间隙，8px 灰色背景 |
| `.preview-phone` | 手机预览外壳，375px 宽，圆角 + 轻阴影 |
| `.preview-phone__content` | 手机外壳内容区 |
| `.dark-strip` | 暗色展示条，用于暗色背景下的组件展示 |

## UI Kit 安全区规则

适用于 `ui_kits/*.html` 中的手机模拟页面（`phone-screen` 内的内容）。

### 顶部状态栏（phone-status）

- `position: absolute` 悬浮在顶部，不占布局空间
- 背景透明，由下方导航栏/页面背景透出颜色
- 高度由 `--safe-area-top` Token 控制（PC 预览 fallback 44px，手机端动态获取）
- 导航栏顶部不内置状态栏间距，由页面外壳统一处理

### 底部 Home 指示器（phone-indicator）

- `position: absolute` 悬浮在底部，不占布局空间
- 背景透明
- 高度由 `--safe-area-bottom` Token 控制（PC 预览 fallback 34px，手机端动态获取）
- 可滚动内容底部安全间距：`--safe-area-bottom-content`
- 底部固定元素（如 bottom-nav、固定按钮）底部 padding 增加 `var(--safe-area-bottom)`

### 安全区 Token

```css
--safe-area-top: env(safe-area-inset-top, 44px);
--safe-area-bottom: env(safe-area-inset-bottom, 34px);
--safe-area-bottom-content: calc(40px + var(--safe-area-bottom));
```

## 布局模式

### 模式 A：通用预览（适用于大多数组件）

使用 `pv-header` + `pv-section` + `.row` + `.label` 组织内容。

```html
<div class="pv-header">
  <h1>组件名称</h1>
  <p>简要描述</p>
</div>

<div class="pv-section">
  <div class="pv-section__title">分区标题</div>
  <div class="pv-section__sub">分区说明</div>
  <div class="row">
    <span class="label">标签</span>
    <!-- 组件实例 -->
  </div>
</div>
```

### 模式 B：列表模拟（适用于 cell、form 等列表型组件）

使用 `.page` + `.section-group` + `.section-gap` 模拟移动端列表页面。

```html
<div class="page">
  <div class="section-group">
    <!-- 列表组件实例 -->
  </div>
  <div class="section-gap"></div>
  <div class="section-group">
    <!-- 另一组列表 -->
  </div>
</div>
```

### 模式 C：聚合验收入口（适用于 `preview/index.html`）

允许使用脚手架区块 + 自定义卡片容器聚合多个单组件预览页，推荐通过 `iframe` 内嵌 `component-*.html`，用于人工验收集中查看。

- 聚合页只负责入口和排布，不复制组件核心示例。
- 聚合页必须明确标注“仅供人工验收查看”，避免被误当成组件契约入口。
- 新增已注册组件时，必须同步把该组件加入聚合页。

## 组件 CSS 标记规则

预览页 `<style>` 块中，组件样式必须用提取标记包裹：

```css
/* @component-css-start */
.tag { ... }
.tag--danger { ... }
/* @component-css-end */
```

- 标记之间只放**组件核心 CSS**。
- 脚手架样式（`.row`、`.label`、`.page`、`.section-group`、`.demo-hint` 等）必须在标记**之外**。
- 脚手架选择器已由 `scaffold.css` 统一提供，预览页不应重复定义。
- `preview/index.html` 这类聚合页不得放 `@component-css-start` / `@component-css-end` 标记，避免被提取脚本误收。

## 使用约束

- `scaffold.css` 仅限 `preview/` 和 `ui_kits/` 目录下的 HTML 文件使用。
- 生产组件不引用 `scaffold.css`，组件样式通过 `components.css` 或内联方式提供。
- 新建预览页时，必须引入 `scaffold.css`，不再自行定义脚手架样式。
- `preview/index.html` 仅作为人工验收聚合入口，不替代 `preview/component-*.html` 单组件预览页。
