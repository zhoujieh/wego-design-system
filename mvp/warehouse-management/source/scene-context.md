# 仓库管理 场景上下文

## 场景入口

- 宿主 Tab：「我的」（my）
- 入口分组：应用中心（my-app-center）
- 入口类型：grid-entry（网格图标入口）
- 入口图标：`./lib/icons/app-center/配货管理.svg`
- 入口标签：「仓库管理」
- 路由方式：hash 路由 `#/my-warehouse-management`
- 进入方式：push（slide-left 动画），覆盖底部 TabBar

## 页面组成

### 1. 仓库列表页（warehouse-page，push 承载）

- 顶部导航栏：左侧返回按钮、中间标题「仓库管理」、右侧「新建」操作按钮
- 摘要区：展示已接入仓库总数、启用中数量、默认发货仓数量
- 仓库列表：卡片模式，每张卡片含仓库封面图（64×64）、名称、状态徽章（默认发货仓/启用中/已停用）、服务范围与发货时效摘要、右侧更多操作按钮（⋯）
- 更多操作菜单：点击更多按钮展开，含「编辑」「删除」两项，删除项为危险色
- 空状态：无仓库时展示图标 + 「还没有仓库」标题 + 引导文案 + 「新增仓库」按钮

### 2. 仓库编辑页（warehouse-editor，full-screen-modal 承载）

新增与编辑共用同一模板，通过 mode 区分：

- 顶部导航栏：左侧「取消」按钮、中间标题（「新增仓库」或「编辑仓库」）、右侧「保存」按钮
- 仓库状态分组：启用仓库开关、默认发货仓开关
- 基础信息分组：仓库名称（必填）、仓库编码、仓库联系人（必填）、联系电话（必填，11 位手机号）、仓库地址（必填）、定位备注、门头图链接
- 履约能力分组：仓库类型（循环选择）、服务范围（循环选择）、支持同城配送（开关）、支持门店自提（开关）、支持直播备货（开关）
- 库存与发货分组：安全库存（数字输入）、单日上限（数字输入）、发货时效（循环选择）、温层要求（循环选择）、锁定预占库存（开关）、跨区域调拨（开关）
- 补充信息分组：仓库备注（多行文本）

### 3. 删除确认弹窗（dialog 承载）

- 标题：「确认删除仓库？」
- 内容：提示删除后无法再选到该仓库
- 按钮：左侧「取消」（dismiss）、右侧「删除」（danger）

## 用户目标

店主集中维护仓库资料，覆盖以下核心操作：

1. 浏览已接入仓库列表，了解仓库总数与启用状态
2. 新增仓库，填写基础信息、履约能力、库存与发货参数
3. 编辑已有仓库，修改资料并保存
4. 删除仓库，通过二次确认避免误操作
5. 切换仓库启停状态与默认发货仓标记
6. 在空状态下快速引导新增第一个仓库

## 主要业务流程

1. **列表浏览**：进入仓库管理 → 查看摘要与卡片 → 浏览仓库详情
2. **新增仓库**：点击「新建」 → 填写表单 → 保存 → 列表回填（新增项置顶）
3. **编辑仓库**：点击卡片更多 → 选择「编辑」 → 修改表单 → 保存 → 列表原位更新
4. **删除仓库**：点击卡片更多 → 选择「删除」 → 确认弹窗 → 确认删除 → 列表移除
5. **空状态新增**：无仓库时点击「新增仓库」 → 填写表单 → 保存 → 列表显示首个仓库

## 业务规则要点

- 默认发货仓互斥：设为默认时自动取消其他仓库的默认标记
- 启停与默认联动：停用仓库时自动取消其默认标记；设为默认时自动启用
- 表单校验：仓库名称、联系人、联系电话（11 位手机号）、地址为必填
- 取消挽留：表单有未保存修改时，点击取消会弹出「放弃未保存内容？」确认
- 数据不持久化：刷新后恢复初始模拟数据（3 条预设仓库）
