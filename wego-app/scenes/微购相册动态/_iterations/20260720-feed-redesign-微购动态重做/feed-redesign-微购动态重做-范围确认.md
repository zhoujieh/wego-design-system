# 微购动态重做范围确认（第 10 版）

## 目标

重构「动态」主 tab 的首屏分发效率与发现流体验，让用户在首屏快速识别并进入感兴趣的动态详情，完整消费文本、图片或视频内容；关联商品详情作为次级链路存在。顶部、卡片信息层级与操作布局必须符合清晰与高效原则。

## 第 10 版相对第 9 版的调整

- 顶部结构：搜索行+tab行+applied行三层 → 单行紧凑顶部（左搜索弱化、右筛选入口）+ 下方一组一级分段切换。
- 内容类型提一级：「全部/产品/笔记」与「推荐/最新」合并为顶部一行一级分段切换（原在二级筛选）；发布者、商品分类仍留二级。
- 卡片布局：自适应等高网格 → 瀑布流双列，卡片按媒体封面自然比例与摘要长度参差撑高。
- 卡片信息层级：类型标签与发布者合并为一行、状态紧凑 inline、操作行一键转发靠右、产品商品入口靠左弱化。
- 首屏密度目标：从「至少两个动态入口」提升到「至少三到四张可识别动态卡片入口」。
- 底部留白：滚动区为底部导航栏与安全区预留，列表底部不被 tabbar 遮挡。
- 筛选联动：选商品分类自动限产品；切笔记自动清商品分类。

## 两类详情的定义与优先级

- 动态详情是首页主要去向：整张动态卡片进入普通二级动态详情页，页面从右侧推入，左上角返回，用于查看发布者、时间以及产品或笔记的完整文本与媒体内容。
- 商品详情是次级去向：只通过动态中的独立商品入口进入普通二级商品详情页，同样使用右侧推入和左上返回，并保持传统电商信息结构，不与动态卡片主点击区混淆。
- 两类详情都是稳定路由的普通二级页，不是全屏 Modal，也不从底部呈现。

## 本次纳入

- 顶部采用单行紧凑结构：左侧全局独立搜索入口（弱化）、右侧集中筛选入口，下方一组紧凑一级切换同时承载「推荐/最新」内容模式与「全部/产品/笔记」内容类型，二者合并为一行分段控件。
- 动态首页使用瀑布流双列卡片：375px 与 393px 常见手机宽度保持双列，卡片按封面自然比例与摘要长度参差撑高；宽屏自动增列、单卡不超过 220px、瀑布流居中。
- 375px 与 393px 宽固定视口下，未滚动前应看到紧凑顶部切换与至少三到四张可识别动态卡片入口，并露出后续内容。
- 瀑布流卡片信息层级紧凑：媒体封面在顶部，下方发布者区只保留一行（头像、名字、产品/笔记类型标签、紧凑 inline 状态），1至2行摘要，底部操作行一键转发靠右、产品商品入口靠左弱化；整张卡片为进入动态详情的主点击区。
- 发布者状态支持正在直播、上新、星标、认证并可并存，首页用紧凑 inline 展示，完整说明放动态详情。
- 全局搜索本期只验证入口位置、识别度与层级合理性；点击后提示「全局搜索能力本期暂未开放」，不下钻。
- 浏览与筛选：顶部一级切换内容模式与内容类型；发布者或店铺、商品分类在集中筛选面板；商品分类只作用于产品，切笔记自动清商品分类。
- 动态详情完整展示全文与全部图片或视频；产品内容额外展示关联商品模块。
- 商品详情传统电商结构，固定底部「联系卖家」「立即购买」本期不下钻，点击分别提示对应能力未开放。
- 从动态详情返回恢复首页内容模式、内容类型、筛选条件与浏览位置；从商品详情返回回到原动态上下文。
- 一键转发在首页靠右、清晰独立但不压过内容入口；在动态详情是唯一强调操作。点击一次直接展示成功反馈。
- 下载图片和更多操作只放动态详情；点击分别提示「图片下载为原型演示，本期不保存到设备」和「更多操作能力本期暂未开放」。
- 所有仅展示不下钻的入口必须立即说明未开放的具体能力，不留无响应、笼统「开发中」或虚假成功反馈。
- 筛选无结果展示空状态并提供清除条件入口。
- 动态首页滚动区底部为底部导航栏与安全区预留足够留白，不被 tabbar 遮挡。

## 本次不做

- 不把商品详情设为动态首页主点击目标。
- 不实现全局搜索的输入、联想、历史记录、结果页和任何下钻交互。
- 不接真实推荐、视频上传、转码或流媒体服务。
- 不进入直播间或实现直播观看与互动，本次只显示正在直播状态。
- 不实现真实联系卖家、购买、加购、询价、下单或支付闭环。
- 不新增好友主页、店铺主页和独立搜索结果路由。
- 不接真实后端、分页、持久化或动态发布。
- 不实现评论、点赞和真实转发发送的业务闭环；一键转发仅模拟成功反馈。
- 不修改消息、工作台和我的 tab。

## 关键路径

1. 进入动态 tab → 浏览瀑布流双列动态卡片 → 点击整张卡片 → 在动态详情查看全文和图片或视频 → 返回并保留首页上下文。
2. 点击顶部全局搜索入口 → 看到「全局搜索能力本期暂未开放」并停留在当前动态页。
3. 用顶部一级切换切换推荐/最新或全部/产品/笔记，或打开筛选面板按发布者或店铺、商品分类筛选 → 查看结果 → 进入动态详情。
4. 进入动态详情 → 查看关联商品 → 进入传统商品详情 → 返回原动态详情。
5. 从动态首页卡片内弱化的商品入口进入商品详情 → 返回原动态上下文。
6. 在动态首页或动态详情点击一次一键转发 → 直接看到成功反馈，当前浏览上下文不变。
7. 在商品详情点击联系卖家或立即购买 → 立即看到对应能力本期暂未开放的提示 → 停留在当前页。
8. 在动态详情点击下载图片或更多 → 分别看到对应提示 → 停留在当前动态详情。

## 原型边界与状态

动态流浏览、组合筛选、动态详情、图片查看、商品详情及返回均需真实可操作；视频播放和一键转发使用本地状态模拟；全局搜索、图片下载、更多操作、联系卖家和立即购买是明确不下钻的入口 stub。覆盖默认推荐、最新、内容类型切换、全局搜索未开放、已筛选、筛选无结果、产品动态详情、笔记动态详情、视频播放、商品详情、转发成功、图片下载原型演示、商品能力未开放、更多操作未开放和宽屏瀑布流自动增列状态。

## 数据与场景语义

原型使用本地发布者、动态、媒体和商品数据。content_type 只使用产品和笔记；两类都必须有文本和媒体列表，并分别覆盖图片与视频媒体。产品必须关联商品并提供 category_id，笔记不要求。发布者 publisher_type=person|shop，筛选时同维度区分个人和店铺。选择商品分类自动限定为产品；切换为笔记自动清除商品分类。发布者状态支持正在直播、上新、星标、认证并可并存。动态详情与商品详情为普通二级场景，共享当前内存上下文。首页卡片目标宽度约 168px 至 220px，宽屏自动增列且保持相同内容层级与点击目标；瀑布流双列参差撑高。
