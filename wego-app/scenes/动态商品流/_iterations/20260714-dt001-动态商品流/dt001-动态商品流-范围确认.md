# dt001 动态商品流 · 范围确认

> 迭代：`dt001`　标题：动态商品流　日期：2026-07-14
> 状态：`awaiting-brief-confirmation`　就绪度：`ready-with-assumptions`

## 一、本次要做什么

让用户在微购相册中连续浏览店主发布的商品动态。每条动态稳定呈现发布者、发布时间、商品内容、商品图片与相关产品信息，并支持进入产品详情或执行必要操作。

## 二、纳入范围

1. 动态商品流浏览场景：连续浏览店主发布的商品动态
2. 每条动态稳定呈现：发布者、发布时间、商品内容、商品图片、相关产品信息
3. 兼容只有产品信息 / 同时含文字 两种动态形态
4. 兼容文字为空、图片数量不同、产品信息较长时的稳定展示
5. 进入产品详情入口
6. 必要操作入口：一键转发（核心操作）、复制文案、分享、收藏、下载图片、搜索、编辑、复制动态、图片点击看大图

## 三、不做什么

1. 发布动态、评论点赞等社交互动
2. 必要操作的具体功能实现（本轮只放入口，不下钻）
3. 真实后端接口与持久化

## 四、从哪里进入

- **动态主 tab**：作为底部导航「动态」tab 的内容，打开 App 即默认进入。店主身份默认可见，App 启动后默认落在该 tab。

## 五、关键路径如何完成

| 路径 | 步骤 | 原型边界 |
| --- | --- | --- |
| 浏览 → 产品详情 | 进入动态流 → 连续浏览动态 → 进入产品详情 | 浏览 functional，产品详情 simulated |
| 浏览 → 必要操作 | 进入动态流 → 连续浏览动态 → 执行必要操作 | 浏览 functional，操作入口 stub |

## 六、原型模拟到什么程度

- **动态商品流浏览场景** functional：可连续浏览、动态状态真实可操作
- **产品详情** simulated：本地模拟数据展示基本产品信息
- **必要操作入口** stub：本轮只呈现入口与明确反馈，不下钻具体功能实现

## 七、业务状态

- `feed_loaded`：动态流加载完成，有动态可连续浏览
- `feed_empty`：动态流为空的边界状态
- `item_with_text_and_product`：同时含文字和产品信息的动态形态
- `item_product_only`：只有产品信息、文字为空的动态形态
- `item_images_varied`：图片数量不同时的稳定展示（0张 / 1张 / 多张）
- `product_detail_entered`：进入产品详情视图，本地模拟展示基本产品信息
- `operation_triggered`：必要操作入口被触发，呈现明确用户可见反馈（stub）

## 八、数据合同

```
dynamic_item:
  id            string，动态唯一标识
  publisher:
    name        string，发布者名称
    avatar      string，发布者头像路径
  publish_time  ISO 8601 datetime，发布时间
  content       string，商品内容文案，可为空
  images        string[]，商品图片路径数组，数量 0-N
  product:
    id          string，关联产品标识
    name        string，产品名称
    price       number，产品价格
    description string，产品补充信息，可选，可能较长
feed:
  items         dynamic_item[]，动态列表
  has_more      boolean，是否还有更多动态可加载
```

## 九、低风险假设（可逆）

1. 假设店主默认已登录，打开 App 即进入动态主 tab
2. 假设动态数据为本地模拟数据，模拟店主已发布的商品动态
3. 假设动态流中的发布者为当前店主相册，浏览者为店主本人
4. 假设「编辑」操作针对当前动态，「复制动态」复制整条动态可转发内容
