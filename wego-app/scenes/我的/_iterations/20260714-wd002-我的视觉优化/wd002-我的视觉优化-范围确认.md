# wd002 - 我的视觉优化 范围确认

> 迭代状态：draft → 等待用户确认 brief
> 主业务场景：我的
> 关联迭代：wd001（已完成，本迭代基于其产物做视觉优化）

## 背景与动机

wd001 已完成「我的」页面的结构、功能入口与数据合同，但视觉层存在多处与微购设计规范不一致的问题，且 15 个应用功能入口图标全部加载失败。本迭代在不改动 DOM 结构、功能入口与数据内容的前提下，按设计规范做视觉优化。

## 目标

按微购设计规范对「我的」页面做全量视觉优化，使其在 Token 合规性、图标可用性、视觉精致度上达标。

## 范围

### 包含

1. **修复 15 个应用功能入口图标加载失败**
   - 现状：scene.js 中 APP_ENTRIES 的 icon 字段使用拼音 ID（如 `icon-yijiankaituan`），实际 SVG 文件全部为中文名（如 `一键开团.svg`），导致 15 个图标全部 404
   - 修复方式：将 icon 字段改为对应中文文件名
   - 「会员管理」无完全匹配资产，选择语义接近的 `粉丝会员卡.svg`
   - 「更多应用」无对应 SVG 文件，改用 iconfont `icon-yingyongzhongxin` class 模式

2. **Token 合规性修正**
   - 硬编码 rgba 数值（如 `rgba(255,255,255,0.15)`、`rgba(255,255,255,0.08)`）替换为合规 Token
   - 按钮、标签、VIP badge、role tag 等深色背景元素统一走 `--bg-inverse-surface-l1`、`--bg-inverse-control` 等 inverse Token（在 sceneTokenPolicy 允许范围内做近似处理）
   - 空间进度条 `height: 6px` 等硬编码数值改用 `--spacer-*` Token

3. **数字字体应用**
   - 订单数量、空间使用量等数字展示使用 WegoKeyboard N9 数字字体
   - 通过 `--number-*` 前缀 Token 或 `font-family` 直引（仅当在场景可消费范围内）

4. **视觉精致度提升**
   - header 区氛围渐变优化
   - 卡片阴影层次（`--shadow-xs/sm/md`）
   - VIP 卡片质感提升
   - 应用图标容器统一规格与背景
   - 配色层次与间距节奏优化

### 不包含

- 新增或删除功能入口
- 修改 DOM 结构、数据字段或交互逻辑
- 创建 DDR（设计系统缺口记录）
- 修改设计系统源文件（`.codex/skills/wego-design/`）
- 新增页面路由或场景

## 边界

- 仅视觉表现层：scene.css 与 scene.js 中视觉相关代码（资源引用、模板字符串中的 class/style）
- 数据结构、功能入口、交互行为完全保持 wd001 现状
- 不创建 DDR，按钮 inverse 变体与 inverse/number Token 缺口在现有可消费 Token 范围内做近似处理

## 影响范围

- `wego-app/scenes/我的/scene.js`（图标资源引用、模板字符串中的视觉 class）
- `wego-app/scenes/我的/scene.css`（Token 合规、阴影、间距、字体、精致度）
- `wego-app/scenes/我的/design-decisions.json`（重提取决策证据）

## 假设

- 本迭代仅修改视觉表现层，不触碰数据/结构/交互
- 用户选择不创建 DDR，设计系统缺口在现有 Token 范围内近似处理
- 「会员管理」可使用「粉丝会员卡.svg」、「更多应用」可使用 iconfont `icon-yingyongzhongxin` 作为视觉替代

## 验收

- 15 个应用功能入口图标全部加载成功（浏览器 Network 无 404）
- 硬编码 rgba 数值已替换为合规 Token
- 视觉精致度提升且不破坏现有功能入口
- 场景合同、交互守卫与固定视口视觉检查通过
