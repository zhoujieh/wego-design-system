# wd001-我的-范围确认

## 目标

提供店主个人中心页面，展示用户信息、会员与空间状态、应用功能入口、订单状态和设置入口，支持快捷操作与导航。

## 范围

### 包含

1. **顶部用户信息区**
   - 左侧大头像
   - 右侧：用户名 + VIP标签 + 角色标签（客服、超级管理员）
   - 底部：+员工按钮、员工管理按钮

2. **会员/空间卡片**
   - VIP年度超级会员标识 + 去续费按钮
   - 空间使用进度条（117.29G/360G）+ 空间管理链接

3. **应用功能入口区**
   - 一键开团、店铺装修、收款码、我的小店、会员管理、访客足迹、数据中心、微信群发、一键换肤、团队管理、专享小程序、相册网址、私域键盘、PC(电脑版)、更多应用

4. **订单状态区**
   - 标题"我买的" + 全部链接
   - 待付款、已付款(16)、已发货、退款、挂起(1)

5. **设置列表区**
   - 设置、我的钱包、我的收藏、我的卡券、提现与返佣

6. 所有入口提供点击反馈（stub），不下钻具体功能实现

7. 页面适配移动端视口（375px/393px）

### 排除

- 各入口的具体功能实现（本轮只放入口，不下钻）
- 真实后端接口与持久化
- 员工管理、空间管理等具体管理功能
- 订单详情与操作

## 入口

作为底部导航「我的」tab 的内容，用户点击底部导航「我的」tab 进入。

## 关键路径

进入我的页面 → 浏览各功能区 → 点击任一入口

## 状态

- profile_loaded：页面加载完成，展示完整用户信息与功能区
- vip_active：VIP年度超级会员标识显示
- space_usage：空间使用进度条展示（117.29G/360G）
- order_counts：订单状态区显示各状态数量（已付款16、挂起1）
- entry_clicked：功能入口被点击，呈现明确用户可见反馈（stub）

## 数据合同

```
user: {
  name: string,
  avatar: string,
  is_vip: boolean,
  vip_level: string,
  role: string
}

space: {
  used: number,
  total: number
}

orders: {
  pending_payment: number,
  paid: number,
  shipped: number,
  refund: number,
  suspended: number
}
```

## 假设

1. 假设用户默认已登录，能正常访问个人中心（low，reversible）
2. 假设页面数据为本地模拟数据（low，reversible）
3. 假设角色标签显示为客服或超级管理员（low，reversible）
