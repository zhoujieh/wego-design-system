# 安全区间距问题修复计划

## 问题概述

从 biz-settings UIKit 页面发现两个安全区相关的布局问题：

1. **顶部重叠**：状态栏（phone-status）与导航栏（navbar）内容重叠，表现为"保存"按钮和右上角"5G"文字叠在一起
2. **底部间距不足**：底部 Home 指示器（phone-indicator）与 bottom-nav 内容间距过近

## 根本原因分析

### 问题 1：顶部重叠

**错误做法**：把 `padding-top: var(--safe-area-top)` 加在了父容器 `.uikit-host-screen` / `.uikit-modal-screen` 上。

CSS 原理：绝对定位元素的 `top: 0` 是相对于**父元素的 padding box**（内容区顶部），而非 border box。当父元素有 `padding-top: 44px` 时：
- StatusBar（`position: absolute; top: 0`）实际渲染在 y=44px 位置
- navbar 作为 flex 第一个子元素，也从 y=44px 位置开始
- 两者完全重叠

**正确逻辑**：
- 父容器不加 `padding-top`
- StatusBar 绝对定位在屏幕最顶部（y=0），背景透明，透出页面背景
- navbar 自身加顶部安全间距（`padding-top` 或 `margin-top`），让内容从安全区下方开始
- navbar 背景透明，与 StatusBar 一起透出页面背景色，视觉上无缝衔接

### 问题 2：底部间距

**现状**：
- `components.css` 中 bottom-nav 已有 `padding-bottom: env(safe-area-inset-bottom, 0px)`
- biz-settings 页面又额外加了 `padding-bottom: var(--safe-area-bottom)`
- 两者叠加导致底部 padding 过大或位置错乱
- phone-indicator 绝对定位在底部，与 bottom-nav 内容的垂直距离需要明确

**正确逻辑**：
- phone-indicator 绝对定位在屏幕最底部（bottom: 0），透明背景，不占位
- 内容滚动区（phone-body）底部留白：`40px + safe-area-bottom`
- 底部固定栏（bottom-nav）底部留白：`safe-area-bottom`（背景延伸到屏幕底部，内容上移）

## 修复步骤

### 步骤 1：修复 biz-settings 页面顶部安全区
- 移除 `.uikit-host-screen` 的 `padding-top: var(--safe-area-top)`
- 移除 `.uikit-modal-screen` 的 `padding-top: var(--safe-area-top)`
- 新增：`.uikit-host-screen .navbar, .uikit-modal-screen .navbar { padding-top: var(--safe-area-top); }`
- 验证：StatusBar 在 0-44px，navbar 内容从 44px 开始，不重叠

### 步骤 2：修复 biz-settings 页面底部安全区
- 移除 `.bottom-nav` 额外的 `padding-bottom: var(--safe-area-bottom)`（components.css 里已有）
- 保留 `.phone-body` 的 `padding-bottom: var(--safe-area-bottom-content)`
- 验证：phone-indicator 横条与 bottom-nav 内容之间有合理间距

### 步骤 3：同步修复 app UI Kit 页面
- 检查 app 页面的顶部安全区实现是否正确
- 检查底部安全区是否正确

### 步骤 4：更新规范文档
- 更新 `预览页脚手架规范.md` 的安全区章节

- 补充示意图或结构说明

### 步骤 5：验证
- 本地静态服务打开 biz-settings 和 app 页面
- 核对顶部：状态栏文字与导航栏内容不重叠
- 核对底部：指示器横条与 bottom-nav 内容间距合理
- 核对滚动内容：底部最后一个元素与指示器之间有 40px 间距

## 风险与注意事项

1. **navbar 背景透明的影响**：navbar 加 `padding-top` 后，透明背景会延伸到 status bar 区域，视觉上与页面背景一致，这是预期的效果。
2. **bottom-nav 的双重 padding**：components.css 已有 `padding-bottom: env(safe-area-inset-bottom, 0px)`，UI Kit 页面不要再重复加，否则会叠加。
3. **modal 页面的背景色**：modal 页面的背景色在 `.uikit-modal-screen` 上，StatusBar 透明后透出的是 modal 的背景色，这是正确的。
4. **与组件规范的一致性**：navbar 组件本身不应内置顶部安全间距（因为它可能用在不需要安全区的场景），安全间距应由页面/容器层负责。
